### **요약: 자바스크립트 라이브러리 및 프레임워크 종류 TOP 10**

이 문서는 한국 실무와 전 세계 개발 생태계에서 널리 사용되는 자바스크립트 기술 10가지를 소개하고 있습니다. 각 기술의 핵심 역할과 특징을 설명하며, 취업 및 실무 활용도를 기준으로 추천하고 있습니다.

  * **주요 3대장:** **React.js**(가장 인기 많음), **Vue.js**(입문 용이), **Angular.js**(구글 주도, 대규모 프로젝트용)
  * **특수 목적 및 차세대:** **Next.js**(React 기반 SSR/SEO 최적화), **Svelte**(컴파일러 기반 고성능), **Remix**(웹 표준 중심 React 프레임워크)
  * **기타 및 레거시:** **jQuery**(구형 프로젝트 유지보수), **Node.js**(서버 사이드 실행 환경), **Three.js**(3D 그래픽), **Nexacro**(국내 SI/기업용 솔루션)

### **상세 조사: 각 라이브러리/프레임워크별 특징 및 장단점**

요청하신 대로 각 항목에 대해 심층 조사한 특징을 정리했습니다.

#### **1. 리액트 (React.js)**

메타(구 페이스북)가 개발한 UI 구축 라이브러리로, 현재 **전 세계 및 국내 점유율 1위**입니다.

  * **핵심 특징:**
      * **가상 DOM (Virtual DOM):** 데이터 변경 시 메모리상의 가상 DOM을 먼저 변경하고 실제 DOM과 비교해 바뀐 부분만 업데이트하여 성능을 최적화합니다.
      * **컴포넌트 기반:** UI를 재사용 가능한 독립 단위(Component)로 쪼개어 개발합니다.
      * **최신 동향:** React 19 및 Server Components 도입으로 서버 측 렌더링 성능이 더욱 강화되고 있습니다.
  * **장점:** 방대한 생태계와 커뮤니티, React Native를 통한 모바일 앱 개발 가능, 취업 시장 수요가 가장 많음.
  * **단점:** 라이브러리이므로 상태 관리(Redux, Recoil 등)나 라우팅 등을 위해 별도 라이브러리를 학습해야 함.

#### **2. 뷰 (Vue.js)**

전 구글 엔지니어 에반 유(Evan You)가 개발한 프레임워크로, **입문 장벽이 낮고 유연**합니다.

  * **핵심 특징:**
      * **진보적 프레임워크 (Progressive Framework):** 프로젝트 규모에 따라 필요한 기능만 점진적으로 도입할 수 있습니다.
      * **단일 파일 컴포넌트 (SFC):** HTML, CSS, JS를 `.vue` 파일 하나에서 관리하여 직관적입니다.
      * **최신 동향:** Vue 3에서 컴포지션 API(Composition API)를 도입하여 대규모 앱에서의 코드 재사용성과 타입스크립트 지원을 강화했습니다.
  * **장점:** React보다 배우기 쉽고 문서화가 매우 잘 되어 있음. HTML/CSS에 익숙한 퍼블리셔나 디자이너와 협업하기 좋음.
  * **단점:** React에 비해 생태계가 작음. 대규모 엔터프라이즈 시장에서는 React나 Angular에 비해 점유율이 낮음.

#### **3. 앵귤러 (Angular)**

구글이 지원하는 **완전한 구조의 웹 프레임워크**입니다.

  * **핵심 특징:**
      * **풀 패키지:** 라우팅, HTTP 클라이언트, 폼 검증 등이 모두 내장되어 있어 별도 설치가 필요 없습니다.
      * **TypeScript 기반:** 기본적으로 타입스크립트를 강제하여 코드의 안정성이 높습니다.
      * **최신 동향:** 최근 'Signals'라는 새로운 반응형 모델을 도입하여 성능과 개발 경험을 개선하고 있습니다.
  * **장점:** 구조가 엄격하여 대규모 팀 프로젝트에서 코드 일관성을 유지하기 좋음. 유지보수성이 뛰어남.
  * **단점:** 배우는 데 시간이 오래 걸림(높은 러닝 커브). 무거운 편이라 초기 로딩 속도가 느릴 수 있음.

#### **4. 넥스트 (Next.js)**

React를 기반으로 만든 **프로덕션 레벨의 프레임워크**입니다. 최근 React 개발의 표준처럼 여겨집니다.

  * **핵심 특징:**
      * **서버 사이드 렌더링 (SSR) & 정적 사이트 생성 (SSG):** 검색 엔진 최적화(SEO)에 불리한 React(CSR)의 단점을 완벽히 해결합니다.
      * **파일 기반 라우팅:** 폴더 구조가 곧 URL 경로가 되어 라우팅 설정이 매우 간편합니다.
      * **App Router:** 최신 버전에서는 리액트 서버 컴포넌트를 적극 활용하는 새로운 라우팅 시스템을 제공합니다.
  * **장점:** 설정이 거의 필요 없이 바로 SSR/SEO 최적화 가능. Vercel을 통한 배포 편의성.
  * **단점:** React를 먼저 알아야 사용 가능. 서버 로직이 포함되므로 배포 환경 구성이 단순 정적 사이트보다 복잡할 수 있음.

#### **5. 제이쿼리 (jQuery)**

2000년대 후반 웹 개발을 지배했던 라이브러리로, 현재는 **레거시 기술**로 분류됩니다.

  * **핵심 특징:**
      * **DOM 조작 간소화:** `$('#id')`와 같은 문법으로 복잡한 자바스크립트 DOM 선택 및 조작을 매우 쉽게 만듭니다.
      * **크로스 브라우징:** 구형 브라우저(IE 등) 간의 호환성 문제를 해결해 줍니다.
  * **장점:** 배우기 매우 쉽고, 기존에 만들어진 수많은 플러그인을 바로 사용할 수 있음.
  * **단점:** 가상 DOM을 사용하는 최신 프레임워크(React, Vue)에 비해 성능이 떨어짐. 대규모 앱 관리에는 부적합.

#### **6. 노드 (Node.js)**

브라우저 밖에서 자바스크립트를 실행할 수 있게 해주는 **런타임 환경**입니다. (엄밀히 말하면 라이브러리가 아님)

  * **핵심 특징:**
      * **비동기 I/O (Non-blocking):** 요청을 멈추지 않고 처리하여 높은 처리량을 가집니다.
      * **이벤트 기반:** 싱글 스레드 모델이지만 이벤트 루프를 통해 효율적으로 동작합니다.
  * **장점:** 프론트엔드 개발자가 같은 언어(JS)로 백엔드 서버까지 개발 가능. 방대한 패키지 매니저(npm) 생태계.
  * **단점:** 싱글 스레드 특성상 CPU 연산이 많은 작업(이미지 처리, 복잡한 계산 등)에는 성능 저하가 발생할 수 있음.

#### **7. 스벨트 (Svelte)**

가상 DOM을 사용하지 않는 **컴파일러 기반의 프레임워크**입니다.

  * **핵심 특징:**
      * **No Virtual DOM:** 실행 시점(Runtime)이 아닌 빌드 시점(Compile time)에 코드를 최적화된 바닐라 JS로 변환합니다.
      * **최신 동향:** Svelte 5에서 'Runes'라는 새로운 반응형 시스템을 도입하여 대규모 앱에서의 상태 관리를 개선했습니다.
  * **장점:** 가상 DOM 비교 작업이 없어 속도가 매우 빠르고 메모리 사용량이 적음. 코드 양이 다른 프레임워크보다 훨씬 적음.
  * **단점:** React나 Vue에 비해 아직 생태계와 라이브러리 지원이 부족함.

#### **8. 넥사크로 (Nexacro)**

국내 소프트웨어 기업 투비소프트가 개발한 **기업용(Enterprise) UI/UX 플랫폼**입니다.

  * **핵심 특징:**
      * **OSMU (One Source Multi Use):** 한 번의 개발로 PC, 모바일, 태블릿 등 다양한 환경에 배포할 수 있습니다.
      * **WYSIWYG 에디터:** 엑셀이나 파워포인트처럼 화면을 드래그 앤 드롭으로 그리는 강력한 전용 개발 도구(Nexacro Studio)를 제공합니다.
  * **장점:** 데이터 그리드(표) 등 기업 업무 화면 개발 생산성이 압도적으로 높음. 국내 공공기관 및 금융권 SI 프로젝트 수요가 많음.
  * **단점:** 전용 도구와 문법을 배워야 하며, 일반적인 웹 표준 생태계와는 거리가 있어 커리어 확장에 제약이 있을 수 있음.

#### **9. 쓰리 (Three.js)**

웹 브라우저에서 **3D 그래픽**을 구현하기 위한 표준 라이브러리입니다.

  * **핵심 특징:**
      * **WebGL 추상화:** 복잡한 WebGL 코드를 직접 짜지 않고도 3D 모델, 조명, 카메라 등을 쉽게 다룰 수 있게 해줍니다.
  * **장점:** 웹페이지에 화려한 3D 인터랙티브 요소를 넣을 수 있는 사실상 표준 도구.
  * **단점:** 3D 좌표계, 렌더링 원리 등 그래픽스에 대한 기본 지식이 필요함.

#### **10. 리믹스 (Remix)**

React Router 팀이 만든 **웹 표준 지향 풀스택 프레임워크**입니다. (최근 쇼피파이에 인수됨)

  * **핵심 특징:**
      * **중첩 라우팅 (Nested Routing):** 라우트별로 데이터를 병렬로 로딩하여 페이지 전환 속도가 매우 빠릅니다.
      * **웹 표준 준수:** Form 태그 등 기본 HTML 동작을 최대한 활용하며, JS가 비활성화된 환경에서도 동작하도록 설계되었습니다.
  * **장점:** 데이터 로딩 시 워터폴(Waterfall) 현상을 제거하여 사용자 경험(UX)이 뛰어남.
  * **단점:** Next.js에 비해 커뮤니티 규모가 작고, 호스팅 및 배포 옵션 선택 시 고려사항이 있을 수 있음.
